# Feature history

Each **`FR-NNNN`** effort may have a directory **`FR-NNNN-<slug>/`** with intake notes, design, **`tickets.md`**, diaries, and **`90-closeout.md`**.

This skeleton ships only **`FR-0000-bootstrap`** as a starting point. Add new feature directories as work grows — use the **`feature-request`** skill (**`.cursor/skills/feature-request/SKILL.md`**) for the full intake → design → tickets → optional **`develop-frontier`** path.

See **`REGISTRY.md`** for id assignment rules.
